Workflow

Discover → coordinate → alert → sync. Built for reliability and iteration.

Peer discovery

Node announce/heartbeat + cleanup to keep a current view of reachable peers.

Chat + updates

Lightweight messaging and posts designed for fast operational coordination.

SOS broadcast

Create and acknowledge emergency alerts to coordinate response across devices.

Offline retention

Mobile caches posts locally with retention cutoffs and expiry for field conditions.

How the mesh works (this repo)

Minimal by design: discover peers, establish links, then forward messages with de-dupe + TTL.

Web (P2P over IP)

Browsers connect via WebRTC data channels. Messages are forwarded gossip-style with TTL for multi-hop delivery.

transport: webrtc • routing: ttl • scaling: bounded-degree

Mobile (P2P over Bluetooth LE)

Phones advertise + scan a shared BLE service, connect, then broadcast message chunks via notifications. Peers reassemble, de-dupe, and forward with TTL.

transport: ble • mtu: 20 bytes • routing: ttl

WAN & rooms

For web WAN connectivity, a room-scoped WebSocket is used for discovery + signaling. TURN is optional for NAT traversal.

signaling: websocket • stun: default • turn: optional

Security (current status)

Transports are real, but end-to-end encryption and authenticated peer identity are not implemented yet.

e2ee: no • identity: no

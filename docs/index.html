Architecture

This project is structured as a powerful pnpm monorepo with distinct applications and shared libraries.

📱 emergency-mesh

React web dashboard. Built with Vite, Tailwind CSS, and Radix UI.

artifacts/emergency-mesh

📲 mesh-mobile

React Native Expo app for field use, background sync, and offline maps.

artifacts/mesh-mobile

⚙️ api-server

Express.js backend handling data sync, WebSockets, and persistence.

artifacts/api-server

📦 Shared Libraries

Drizzle ORM db, Zod schemas, React Query clients, and OpenAPI specs.

lib/*

Tech Stack

  • Frontend: React, React Native (Expo), Tailwind CSS, Radix UI, Vite
  • Backend: Node.js, Express.js, WebSockets (ws)
  • Database & State: Drizzle ORM, React Query
  • Tooling: TypeScript, pnpm (Workspaces), ESLint, Prettier

Setup Instructions

Prerequisites

  • Node.js (v18 or higher recommended)
  • pnpm (v8 or higher)

Installation

git clone <repository-url>
cd mesh-network
pnpm install

Running the Services

Start the API Server:

pnpm --filter @workspace/api-server run dev

Start the Web Dashboard (available at http://localhost:5173):

pnpm --filter @workspace/emergency-mesh run dev

Start the Mobile App (use Expo Go to scan QR):

pnpm --filter @workspace/mesh-mobile run dev

How to Modify and Improve

Database Models

Define your new schema in lib/db/src/schema/, update the export in lib/db/src/index.ts, generate migrations, and utilize the new tables in the API server.

Expanding the API

Update the OpenAPI spec in lib/api-spec/openapi.yaml, generate types, implement routes in artifacts/api-server/src/routes/, and add validation to lib/api-zod.

UI Enhancements

Modify shadcn/ui components in artifacts/emergency-mesh/src/components/ui/ or update mobile components in artifacts/mesh-mobile/components/.

Network & Offline Capabilities

Enhance background sync in artifacts/mesh-mobile/services/backgroundSync.ts and update real-time node coordination in artifacts/api-server/src/mesh.ts.
